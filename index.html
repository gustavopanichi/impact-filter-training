Impact Filter
PM Training Guide

A step-by-step guide to using the intake assessment tool. Paste a brief, get a structured read. You make the call.

Get started ↓
01 — Setup

You need to have Claude first

The Impact Filter runs inside a Claude Project. Here's how to get in.

1

Get a Claude account

Go to claude.ai and log in or sign up. A free account works. If your team has a Claude Team or Enterprise workspace, use that — your conversations are private to you either way.

2

Open the Impact Filter project

In Claude, look for Impact Filter in your sidebar under Projects. If you don't see it, ask your team lead to share it with you. The project has all the instructions and account data baked in.

3

Start a new conversation

Click New chat inside the project. Each new intake should be its own conversation so things stay clean and findable.

4

That's it

No plugins. No extensions. No configuration. Paste your intake material and the tool gives you a structured assessment like this:

Important: If you paste a brief into a regular Claude chat (not the project), it won't know the Impact Filter framework, the account team structure, or the SME logic. Always use the project.
02 — How to use it

Paste anything. Get a structured assessment.

The tool reads messy inputs and returns clean, scannable analysis.

Works great

Client emails, Slack messages, brief docs, call notes, meeting summaries, intake forms, forwarded threads.

Also works

Partial info. "McDonald's wants two slide guides, ~42 slides, due end of March." The tool flags what's missing.

Doesn't work

Empty messages with no project details. The tool needs something to analyze — give it the raw material.

File attachments

Drag and drop PDFs, Word docs, or PowerPoints into the chat. The tool reads them directly.

Example intake

You paste: "Got an email from Spotify. They want visual assets for a press feature — celebrating a couple's relationship through shared listening history. Need a data viz, custom album cover, and social/media assets. EVB, SJR, and Spotify editorial are involved. No budget or timeline mentioned yet."

You get: A full Impact Filter assessment — account team (DD: Gustavo, PM: Matthew), gap analysis (budget ❌, timeline ❌, visual direction ❌), risk flags, Path B recommendation, and copy-ready questions for the client.

Follow-ups work too. After the initial assessment, add info in the same chat: "Budget confirmed at $15k, deadline is May 15th." The tool updates its read. Ask it to regenerate the full summary if you want a clean version.
03 — Reading the output

What the assessment looks like

Every assessment has two parts: a quick-scan summary header and a detailed breakdown.

Summary header

The 10-second read. Client name, project description, type of work, recommended path (A/B/C with one sentence why), account team (DD + PM), and any SMEs flagged. If any field shows ⚠️, confirm it yourself.

What we know / what's missing

Six fields checked: budget, visibility, deliverables, timeline, clarity, capabilities. Each gets ✅ confirmed, ⚠️ provisional, or ❌ missing.

Risk flags

🔴 = confirmed. 🟡 = provisional. Triggers are strategic concerns (high-impact visibility, ambiguous direction, brand-defining). Complexity flags are operational (unclear scope, constrained timeline, cross-functional).

Consider consulting

The account DD is always first. SMEs appear only when specific triggers fired. Each entry includes the key question to ask yourself.

Questions to resolve + next steps

Client-ready questions you can copy-paste. Plus 2–4 action items. Only covers what's actually missing.
04 — Paths

Path A, B, or C

Every assessment recommends one of three paths. This is a suggestion — you make the final call.

Path A

Standard execution

The brief is clear. The work follows an existing template or system. No big decisions needed. Just execute.

Typical examples: Resize assets in existing template. Reformat a deck. Minor content swaps with clear instructions. Export or file clean-up.
Path B

Brief consultation

Mostly executable, but needs a quick PM/DD/PD alignment first. Higher impact or more complex than Path A. A short check-in saves rework.

Typical examples: New asset type inside an existing system. Short timeline with unclear audience. Mixed stakeholder input on a doable project.
Path C

Full strategic engagement

Big decisions before design starts. Net-new brand, narrative, positioning, or concept work. A creative lead shapes direction first.

Typical examples: Full rebrand. Executive keynote with unclear message. Flagship campaign. Foundational template system.
Default is A. The tool only recommends B or C when it detects specific triggers. If it goes higher, read the "why" — it's flagging something concrete.

06 — Tips and gotchas

Things to keep in mind

Practical advice and common misunderstandings.

More context = better output

Richer input means fewer provisional flags. Paste the client email and your internal notes together.

One project per chat

Don't paste three intakes into one conversation. Start fresh for each project.

Copy the questions

"Questions to resolve" is client-friendly on purpose. Copy them straight into your next email.

You decide, always

The tool advises. If it says Path B and you think A, go with your judgment. You know the account.

Provisional doesn't mean wrong

🟡 flags and ⚠️ markers mean the intake didn't mention something. It's saying "check this," not "there's a problem."

"External" doesn't mean "high-impact"

Public-facing or social work isn't automatically high-impact. That term means: C-suite audience, full rebrand, flagship campaign, major conference launch.

The six basics always get checked

Budget, visibility, deliverables, timeline, clarity, capabilities. Every intake — even incomplete ones — gets assessed against all six.

You can update the assessment

Got new info? Paste it in the same conversation. The tool adjusts. Say "regenerate the full summary" for a clean version.

07 — FAQ

Common questions

Answers to the things PMs actually ask.

What if the client isn't in the account list?+
The tool flags it as an unknown account and notes the DD needs to be confirmed. It still runs the full analysis.
Can I run the filter on something I already scoped?+
Yes. It's a useful sanity check. Paste the brief and see if the tool flags anything you missed.
What if I disagree with the path recommendation?+
Go with your judgment. You can ask "Why did you recommend Path B?" and it'll explain.
What if I get new info after the assessment?+
Add it in the same conversation. The tool updates. Ask to regenerate the summary for a clean version.
Can I use this for small or simple projects?+
Yes. Simple projects get Path A, no flags, 10 seconds. Still useful as a checkpoint.
Who can see my conversations?+
Only you, unless you share them. Claude Projects keep conversations private to the individual user.
What are "the six basics"?+
Budget, visibility, deliverables, timeline, clarity, and capabilities. Each gets ✅ confirmed, ⚠️ provisional, or ❌ missing.
